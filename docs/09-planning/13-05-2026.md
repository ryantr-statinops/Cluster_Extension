# Docs Restructure Execution Plan

# Objective

Mục tiêu của task này là:

- restructure toàn bộ thư mục `docs/`
- chuyển docs hiện tại thành:
  - AI-readable architecture system
  - Obsidian-compatible knowledge graph
  - scalable documentation structure
- normalize lại structure
- tạo cross-link giữa các docs
- chuẩn bị nền tảng cho implementation phase

---

# IMPORTANT EXECUTION RULES

## Rule 1 — Read Everything First

PHẢI:
- đọc toàn bộ markdown files hiện có trước khi sửa bất kỳ thứ gì
- hiểu nội dung và responsibility của từng file
- detect duplicated concepts
- detect missing concepts

KHÔNG được:
- move files trước khi đọc

---

## Rule 2 — Preserve Existing Content

KHÔNG được:
- xóa nội dung cũ
- rewrite architecture logic lớn
- simplify concepts

Được phép:
- reorganize
- normalize heading
- add references
- split sections nếu cần

---

## Rule 3 — Build Knowledge Graph

Tất cả markdown files:
nên được linked bằng Obsidian wiki links.

Ví dụ:

```md
[[pipeline-runtime]]
[[entities]]
[[zustand-architecture]]
[[react-flow-architecture]]
````

---

## Rule 4 — Respect Ownership Boundaries

Mỗi file:
phải có responsibility rõ ràng.

Ví dụ:

* runtime docs
  ≠ frontend docs

* state docs
  ≠ persistence docs
